**Key Technical Concepts**
-   **Dependency Locking:** The critical importance of not altering  without specific intention () to prevent breaking changes from transitive dependencies, especially to avoid browser-specific bugs like Safari's Temporal Dead Zone (TDZ).
-   **SOC 2 Compliance:** Implementing key controls like Role-Based Access Control (RBAC), separation of duties (Founder vs. Operator), and maintaining an immutable audit trail for all privileged actions.
-   **Architectural Gating:** Using a combination of frontend components () and authoritative backend API checks to robustly enforce business logic and security rules.
-   **Cascade Deletes:** The process of safely deleting a primary record (e.g., a user account) and ensuring all its dependent data across multiple database collections is also removed, while correctly preserving linked but independent records (e.g., other user accounts).
-   **Dual-Role User Model:** Architecting the system to allow a single user account to hold multiple roles (e.g., being both a beneficiary of one estate and a benefactor of their own) and switch between contexts seamlessly.
